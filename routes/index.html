1. Perbandingan Model Pengembangan Perangkat Lunak

Model Karakteristik Utama Kelebihan Kekurangan Cocok Untuk
Waterfall Sequential, linear, fase berurutan tanpa overlap • Mudah dikelola dan dipahami
• Dokumentasi lengkap
• Timeline jelas
• Cocok untuk proyek besar 		• Tidak fleksibel terhadap perubahan
• Testing di akhir
• Sulit kembali ke fase sebelumnya
• Risiko tinggi jika requirement salah 		Proyek dengan requirement stabil, well-defined, dan tidak ada perubahan yang diharapkan
Incremental Kombinasi waterfall dan iterative, pengembangan bertahap • Dapat memberikan hasil partial
• Risiko lebih rendah
• Feedback lebih cepat
• Testing paralel dengan development 		• Butuh planning yang baik
• Arsitektur harus matang dari awal
• Integration complexity
• Dokumentasi dapat terfragmentasi 		Proyek besar yang dapat dibagi menjadi modul-modul independen
RAD (Rapid Application Development) Pengembangan cepat dengan prototyping dan reuse komponen • Development time singkat
• User involvement tinggi
• Feedback cepat
• Mengurangi biaya development 		• Membutuhkan developer berpengalaman
• Tidak cocok untuk sistem besar
• Dependency pada tool dan teknologi
• Dokumentasi minimal 		Sistem dengan timeframe pendek, requirement yang dapat berubah, dan tim kecil
Prototype Membangun model awal untuk validasi requirement • User dapat melihat sistem lebih awal
• Mengurangi risk dan uncertainty
• Identifikasi requirement yang miss
• Improved system usability 		• User bisa expect prototype = final product
• Developer mungkin shortcut dalam prototype
• Time consuming jika berlebihan
• Bisa mengurangi focus pada quality 		Sistem dengan requirement yang tidak jelas atau inovatif dan interaktif
Agile Iterative, adaptive, collaborative, working software focus • Sangat adaptif terhadap perubahan
• Customer collaboration tinggi
• Working software delivered frequently
• Team motivation tinggi 		• Dokumentasi minimal
• Sulit untuk predict timeline/budget
• Membutuhkan customer involvement tinggi
• Tidak cocok untuk semua jenis proyek 		Proyek dengan requirement yang berubah-ubah, tim kecil, dan customer yang aktif
